1. 全局错误捕获 (Global Errors)

window.onerror + unhandledrejection 自动捕获,链式调用不吞宿主 handler。

2. 网络错误拦截 (Network Errors)

fetch / XHR 拦截器捕获 HTTP 4xx/5xx 和网络失败。SDK 自身请求(/chat/、/auth/)会被排除。

3. DOM 错误弹窗检测 (MutationObserver)

监听 document.body 子树变化,匹配主流 UI 框架的错误组件。SDK 自身 DOM 被排除。

4. 脱敏 / 去重 / 缓冲管理

默认脱敏:JWT token、密码、邮箱、信用卡号自动替换为 [REDACTED]。同一错误 5 秒内不重复入库。

5. AI 上下文注入验证 (端到端)

操作流程:先点下方按钮触发错误 → 打开浮窗发消息(如"帮我录一单") → 观察 AI 是否主动提及页面异常。

点击后会自动触发: fetch 500 + JS Error + DOM Toast,并自动打开浮窗。你只需输入任意消息发送,观察 AI 回复。

6. 表单校验引擎 (validateTool)

纯前端执行,AI 在 submit_form 前调 validate_form 主动校验。支持 5 种规则类型 + error/warning 双严重级。

每个按钮会用预设 formData 直接调 AIAgent.validateTool 工厂产出的 onCall,在日志区显示结果。

8. 浮窗多页面导航 (侧边栏)

浮窗左侧 icon 栏:💬 对话 / 🧠 记忆 / ⚙️ 设置 / 📜 历史(v1 不开)。点击切换,消息历史保留。

9. 快捷指令 (Quick Commands)

输入框输入 / 触发模糊匹配下拉框,键盘 ↑↓ 导航、Enter 选中、Esc 关闭。

内置指令(自动注册):/new(新会话) / /clear(清空) / /help(列出指令)

第三方指令可通过 init({ quickCommands: [...] }) 或 agent.registerCommands([...]) 注入。
